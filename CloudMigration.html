Cloud Migration

Many business are on a journey to embrace the public Clouds, be it AWS, Azure or Google Cloud Platform (GCP).

Any use of the Cloud, be it hybrid, or a Cloud native solution, requires strong access protection. This is best provided by ensuring all access is controlled through an Identity and access manager (IAM). Once this is in place, the migration of on premises capability or the implementation of new applications becomes a relatively straight forward process.

Moving to the Cloud will almost certainly save money and move your IT spend form Capital Expenditure to Operational Expenditure. However, at a technical level there are a number of critical decisions that need to be made.

Today, it is unlikely that an organisation would simply reimplement a set of virtual servers in the Cloud; there are better ways to achieve the desired out-comes.
